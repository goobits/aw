# Frontend Policy

Use the local project's framework, component library, icon system, and styling
conventions from `.agents.local/project.md`.

Shared frontend baseline:

- Build the actual usable experience, not a marketing placeholder, unless the
  user explicitly asks for a landing page.
- Match the existing design system and interaction patterns.
- Prefer familiar controls: icons for tools, segmented controls for modes,
  toggles for binary settings, sliders/inputs for numeric values, menus for
  option sets, and tabs for views.
- Keep layouts responsive and verify text does not overflow or overlap.
- Avoid decorative clutter that does not help the user complete the task.
- Use visual assets for websites, games, branded pages, or object-focused
  experiences when the user needs to inspect or understand the subject.
- For games or specialized interactive tools, prefer proven domain libraries for
  rules, parsing, physics, or AI when appropriate.
